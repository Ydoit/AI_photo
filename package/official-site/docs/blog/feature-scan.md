# 功能全景速览：TrailSnap 能做什么？

大多数人拍照很勤快，整理很随缘：手机相册越堆越满，回忆却越来越难找。你可能记得“那次旅行很开心”，但你很难在几秒钟内翻出“那张在车窗边的照片”、那段行程的时间线、以及一年里你到底去了多少座城市。

TrailSnap（行影集）想解决的就是这件事：把“照片”从文件堆里拎出来，重新变成可以回顾、可以检索、可以统计、可以讲述的旅程。

这篇文章不讲宏大叙事，只做一次“扫功能”：TrailSnap 现在能做什么、它的优势是什么、你可以怎么用它开始第一天。

## 1. 智能相册：从“能看”到“能找”

TrailSnap 的第一层体验，是把你的照片库变成一个更好用的相册：

- **瀑布流浏览**：更适合大量照片的快速浏览与筛选。
- **相册管理**：从“按时间堆”进化到“按主题整理”。
- **位置解析**：把拍摄信息与地理位置关联起来，让照片回到“发生地”。

它不是在做一个“看图工具”，而是在做一个“回忆的索引”。

## 2. AI 能力：让照片拥有“可计算的语义”

照片之所以难整理，是因为它们本质上是“非结构化数据”。TrailSnap 把关键能力拆成了几块，逐步把照片变成结构化信息：

- **OCR 识别**：从照片里读出文字信息（票据、门票、告示牌等）。
- **人脸检测与特征抽取**：把“人物”作为可检索维度，为后续人物聚类与相册组织打底。
- **智能分类**：把照片按内容归类，减少手工打标签的负担。

这些能力的价值不在于“识别本身”，而在于它们会沉淀成可检索字段、可统计指标和可复用的任务结果。

## 3. 行程票据：把“碎片证明”变成“旅程记录”

旅行中最容易被忽略的，是那些散落在相册里的票据截图：火车票、景区票、演唱会门票……它们记录的是“你到过哪里、何时出发、何时抵达”。

TrailSnap 的一个特色方向，是把这些票据从“照片”还原成“行程数据”：

- **火车票信息管理**：支持创建、编辑、删除与列表展示。
- **票据信息自动识别**：通过 OCR 把关键信息提取出来，减少手工录入。

这条路的终点不是“多一个票夹”，而是让照片、票据与轨迹共同构成一条可回顾的时间线。

## 4. 数据可视化：让一年变得可复盘

当照片与行程被结构化之后，可视化就不再是锦上添花，而是“把生活重新讲一遍”的入口：

- **出行统计图表**：城市、时间、频次等维度的复盘。
- **时间轴与线路里程**：让旅程的连续性被看见，而不是碎片化的九宫格。

如果你喜欢“年度总结”，那 TrailSnap 的年度报告会是你打开应用的一个强理由。

## 5. 年度报告：把回忆做成一份“能分享的作品”

TrailSnap 提供自动生成年度报告的能力，目标是把你一年里的出行数据做成一份可以回顾、也可以分享的“作品式总结”，比如：

- 照片墙
- 出行城市与景点
- 行程时间轴
- 线路里程

它更像“自动剪辑的相册回忆”，而不是一页冷冰冰的报表。

## 6. 部署与数据：为什么说“数据真正属于你”

TrailSnap 支持 Docker Compose 一键部署，把前端、后端、数据库与 AI 服务一起拉起。对普通用户来说，它意味着：

- 你的照片仍然在你自己的存储里（本地/家用 NAS）。
- 相册索引、识别结果与统计数据也在你自己掌控的数据库里。
- 你可以按自己的节奏升级、备份、迁移，而不是被平台“下架”或“规则变更”。

如果你计划在 NAS 上部署，可以从用户指南的 Docker 部署章节开始：[/docs/guide/docker/](/docs/guide/docker/)

## 7. 30 分钟入门：建议你从这三件事开始

如果你第一次接触 TrailSnap，可以按下面的顺序体验：

1. **先跑起来**：按安装指南完成 Docker 部署，确认能打开前端页面。
2. **先接入照片目录**：把你的照片目录挂载到服务里，让索引任务有数据可跑。
3. **先做一次回顾**：用统计与时间轴找回“这一年到底去了哪里”。

TrailSnap 的长期目标，是让相册不再只是“存储”，而是你家庭 AI 数据中心里最重要的一块“记忆索引”。它会随着你照片的增长、以及 AI 能力的增强，变得越来越值钱。
