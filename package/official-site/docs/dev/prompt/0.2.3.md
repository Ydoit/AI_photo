图片文件过滤功能：
1. 新增设置项，用户可以自己配置要过滤的文件规则，例如：过滤掉文件小于100kB的图片，图片分辨率小于一定值的图片，文件名符合一定规则的图片等。
2. 新增或删除规则后，触发过滤操作，将符合条件的图片文件从数据库中删除。

前端功能：
1. 在package\website\src\views\settings\ExternalGallery.vue中新增一个“图片文件过滤”选项卡，用户可以在其中配置要过滤的文件规则。
2. 新增或删除规则后，触发过滤操作，将符合条件的图片文件从数据库中删除。
3. 新增一个“应用过滤”按钮，点击后触发过滤操作，将符合条件的图片文件从数据库中删除（不要删除原文件）。
4. 注意主题切换、深浅主题、移动端界面适配

后端实现：
1. package\server\app\core\config_manager.py 新增过滤规则相关配置
2. package\server\app\service\tasks\scan.py 扫描文件夹时应用过滤规则
3. 新增一个过滤规则的api，用户可以通过api新增或删除过滤规则，并在后台触发过滤操作。

支持筛选相册：
1. package\website\src\views\PhotosPage.vue 新增一个筛选照片的按钮，用户可以根据类型进行筛选，默认显示所有照片。
2. 筛选条件包括：
   1. 来源（ImageType（相机、截屏、其他）），选择相机时，还可以根据make+model进行筛选。
   2. 类型（图片、视频、实况图）
   3. 时间（年份）
   4. 地点（城市）
3. Ui设计：筛选项按维度分组展示，每组包含组标题和标签式选项按钮，组标题左对齐，选项按钮横向排列，支持换行。同一分组内支持多选（如 “来源” 可同时选中 “相机” 和 “截屏”），不同分组的选中状态相互独立。点击选项可切换 “选中 / 未选中” 状态，视觉反馈清晰。
4. 筛选条件从后端api获取，包括所有可选的年份、所有可选的城市、所有可选的make+model等。
5. fetchTimelineStats接口新增筛选条件参数，用于根据用户选择的筛选条件，返回符合条件的照片数据。

后端api：
1. 新增一个获取所有筛选条件的接口，用于在前端下拉框中显示所有可选的筛选条件。
2. /api/stats/timeline 新增筛选条件参数，用于根据用户选择的筛选条件，返回符合条件的照片数据。
3. /api/photos 新增筛选条件参数，用于根据用户选择的筛选条件，返回符合条件的照片数据。
