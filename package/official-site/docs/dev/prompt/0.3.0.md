相册实现登录功能和权限管理功能：
一、 核心目标（前置定义）
- 验证用户身份的合法性，仅授权用户可访问相册应用；
- 登录后关联用户专属数据（如个人相册、收藏、权限配置），实现多用户数据隔离
二、 基础登录模块（核心交互）
2.1 登录入口与表单
   - 应用启动后默认展示登录页面，包含清晰的 “账号”“密码” 输入框，以及 “登录”“忘记密码”“注册”（可选）按钮；
   - 输入框支持基础校验：账号非空、密码长度≥6 位，输入错误时实时提示（如 “密码长度不能少于 6 位”）；
   - 适配多端访问场景（网页端、移动端、桌面端），登录表单布局自适应不同屏幕尺寸。
2.2 登录验证逻辑
  - 本地验证：先校验账号 / 密码格式（减少无效请求），再向后端数据库发起验证请求；
  - 身份匹配：验证通过后，返回用户唯一标识、用户名、用户角色（普通用户 / 管理员）；
  - 会话管理：登录成功后生成本地会话（如 Cookie、Token），会话有效期可配置（默认 7 天），会话未过期时再次打开应用无需重复登录；
  - 支持 “记住密码” 功能（可选）：用户勾选后，本地加密存储账号密码，下次打开自动填充；
  - 支持 “自动登录” 功能（可选）：记住密码基础上，下次启动应用直接完成登录。
三、 用户身份区分与数据隔离（核心价值）
- 登录后基于token加载专属数据：每个用户仅能查看 / 编辑自己上传的相册、收藏的照片、自定义的相册分类；
- 管理员角色（特殊用户）：可额外查看所有用户的账号信息、存储空间使用情况，支持添加/删除普通用户账号；
- 数据隔离规则：用户上传的照片默认仅自己可见，可配置 “共享相册”（指定用户可查看），也可配置 “公共相册”（无需登录可查看）。
- 配置隔离，每个用户的配置文件（如基础设置、偏好设置、外置图库）默认仅对该用户可见。
四、 安全防护
- 密码安全：存储到数据库的密码需通过不可逆加密（如 bcrypt、Argon2）处理，禁止明文存储；
- 登录限制：连续输错密码≥5 次时，锁定账号 10 分钟（时间可配置），并提示 “账号已锁定，请 10 分钟后重试”；
- 会话安全：会话 Token 仅存储在本地，退出登录时立即销毁 Token；
- 防暴力破解：记录登录 IP 和次数，对高频异常登录的 IP（如 1 分钟内登录≥10 次）临时封禁）；
五、 异常处理与用户提示
- 登录失败提示：区分 “账号不存在”“密码错误”“账号已锁定”“会话过期请重新登录” 等明确提示（避免笼统的 “登录失败”）；
- 网络 / 数据库异常：数据库未启动时，提示 “服务暂不可用，请检查服务状态”；
- 退出登录：提供明确的 “退出登录” 按钮，点击后销毁会话、清空本地缓存，返回登录页面；
- 忘记密码：支持本地重置（管理员可重置普通用户密码，普通用户可通过预留的密保问题 / 邮箱重置）。

具体设计方案
##### 极简现代风
- **外框**：浅灰色/米白色细边框（10–20px），圆角 8–12px，和界面顶部导航栏风格统一。
- **内边距**：照片与外框之间留 20–40px 空白，避免贴边。
- **背景填充**：
  - 竖图/窄图：左右用**半透明磨砂白/浅灰**填充到全屏宽度。
  - 横图/宽图：上下用同色填充到全屏高度。
  - 也可以提取照片主色调做渐变背景，更柔和。
- **适配逻辑**：
  ```
  照片最大显示宽度 = 屏幕宽度 - 2*外框宽 - 2*内边距
  照片最大显示高度 = 屏幕高度 - 2*外框宽 - 2*内边距
  按原比例缩放，居中放置，剩余区域用背景填充
  ```
