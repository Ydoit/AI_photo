天地图瓦片资源缓存实现：
1. 使用天地图api有次数限制，为了避免重复请求，需要缓存已经请求过的瓦片资源。
2. 部署到docker容器中，使用nginx进行转发，修改前端和nginx配置文件，将天地图的瓦片请求转发到nginx，nginx缓存命中后直接返回缓存的瓦片资源，否则转发到天地图api请求瓦片资源，并缓存到nginx。
3. 本地开发环境下，没有nginx进行转发，直接请求天地图api，没有缓存功能。
