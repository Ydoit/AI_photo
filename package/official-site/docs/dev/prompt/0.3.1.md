### 登录页设计

#### 一、设计主题
趣味互动式登录界面，通过卡通角色与用户操作的联动，提升登录过程的趣味性和用户体验。

#### 二、界面布局
移动端只显示右侧区域
1. **左侧区域**：
   - 包含4个几何形态的卡通角色，分别为：
     - 紫色高长方形角色，带有简单眼睛表情；
     - 黑色中长方形角色，带有眼睛表情；
     - 橙色半圆形角色，带有眼睛和嘴巴表情；
     - 黄色半圆形角色，带有眼睛和嘴巴表情。
   - 背景为浅灰色，角色之间有层次堆叠，整体风格扁平卡通。
2. **右侧区域**：
   - 背景为白色，包含登录表单元素：
     - 标题：“登录 TrailSnap”，字体简洁现代，字号较大；
     - 输入框：用户名输入框、密码输入框，样式简洁，带有焦点状态动效；
     - 功能选项：“记住我”复选框，“忘记密码”；
     - 操作按钮：“登录”按钮（突出显示）；

#### 三、动效设计
- 当用户鼠标移动时，左侧卡通角色会看向鼠标位置；
- 当用户点击用户名输入框时，左侧卡通角色做出眼神看向输入框的动效；
- 当用户在密码输入框输入内容时，卡通角色做出向左看（不偷看密码）的动效；
- 按钮点击时，对应附近的卡通角色做出庆祝类小动作（如小跳一下）。

#### 四、色彩规范
- 中性色：浅灰（#E5E7EB）、白色（#FFFFFF）、黑色（#111827）；
- 卡通角色色：紫色（#6366F1）、黑色（#1E293B）、橙色（#F97316）、黄色（#FACC15）；

1. 紫色和黑色卡通角色高一些，紫色最高最宽

## 相似照片清理功能

### 功能描述

根据image_vectors表中的向量相似度，按相似度聚类，相似照片为一组。用户可以查看所有照片分组，也可以查看某个分组中的照片。可以选择删除某个分组中的照片（保留一张），或者删除所有分组中的照片（每个分组保留一张）。

### 前端界面

1. package\website\src\layouts\NavBar.vue 导航栏“更多”添加一个子项“工具箱”，点击后进入工具箱界面。
2. 把package\website\src\views\HomePage.vue中的“低分清理”功能移动到工具箱界面
3. 工具箱界面新增一个“相似照片清理”功能，点击后进入相似照片清理界面。
4. 相似照片清理界面分组组展示所有照片，组与组之间垂直布局，组内照片水平展示（包含照片和文件名），左右有滚动按钮，点击可以水平滚动。
5. 点击照片可以查看照片详情，上一张、下一张按钮只能在当前分组内切换。
6. 每个分组有一个全选按钮，点击可以选择该分组内的照片（保留第一张）。
7. 右上角一个“删除所有分组”按钮，点击可以删除所有分组中的照片（每个分组保留第一张）。
8. 如果分组很多，注意显示性能
9. 空状态处理：无相似照片分组时，显示 “暂无相似照片分组”，并提示 “可调整相似度阈值重新聚类”；分组内照片全部删除后，自动移除该分组。
10. 加载 / 异常状态：聚类计算中显示加载动画（比如 “正在计算相似照片分组，请稍候...”）；接口请求失败时，显示错误提示（如 “加载相似照片分组失败，请重试”），并提供 “重试” 按钮。
11. 操作反馈：删除操作后，显示提示（如 “成功删除 X 张相似照片”）；滚动按钮交互：组内照片滚动到最左 / 最右时，禁用对应方向的滚动按钮（避免无效点击）；
12. 响应式适配：补充移动端适配规则（比如移动端组内照片改为垂直布局，滚动按钮改为滑动手势）；小屏设备下，照片文件名自动换行或省略显示（避免超出容器）。

### 相似照片清理功能优化

现在相似性聚类结果存在了tasks表中，不方便查看和管理。需要将聚类结果存储在数据库的两个新表中，一个是类别表（image_clusters），一个是照片-类别关联表（photo_clusters）。

聚类结果存储在数据库的两个新表中，一个是类别表（image_clusters），一个是照片-类别关联表（photo_clusters）。
类别表（image_clusters）：
- cluster_id：聚类ID，用于唯一标识一个聚类组
- task_id：任务ID，引用tasks表中的任务ID
- cluster_type：聚类类型，字符串类型，用于描述聚类组的类型（相似性聚类，md5相同聚类）
- count：聚类组中的照片数量，整数类型，记录聚类组中的照片数量
- created_at：创建时间，记录聚类组的创建时间

照片-类别关联表（photo_clusters）：
- id：主键，自增整数
- photo_id：照片ID
- cluster_id：聚类ID，引用image_clusters表中的聚类ID
- created_at：创建时间，记录照片所属聚类组的时间

删除照片时，需要删除照片-类别关联表中对应的记录，同时更新类别表中的count字段，count为1或者0时，需要删除该聚类组。

前端根据任务ID查看聚类结果，可以有多个聚类任务，每个任务有一个唯一的任务ID。
可以查看某个聚类任务的进度
